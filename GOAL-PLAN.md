# SnapDiff - 项目开发计划

## 1. 项目概述
**SnapDiff** 是一个轻量级、通用的文件快照对比与变更日志（Changelog）自动生成工具。
旨在解决传统版本控制系统（如 Git）在处理包含大量二进制文件（如游戏模组、资源包）的项目时，体积臃肿、提交记录繁琐的问题。

## 2. 核心目标
- **极速快照**：一键扫描目录，计算文件 Hash，生成/对比快照表。
- **智能分类**：自动区分二进制大文件（提取元数据如版本号）和文本小文件（记录 Diff）。
- **人机协作**：自动列出文件变动，通过 CLI 交互引导用户补充"人话"描述（如修复的 Bug）。
- **通用性**：不局限于特定游戏或环境，通过配置文件适配各种项目。

## 3. 核心工作流
1. **扫描与 Hash**：递归扫描目标目录，过滤忽略项，计算 MD5/SHA256。
2. **差异对比**：与历史快照对比，得出 增(Add)、删(Delete)、改(Modify)、重命名(Rename)。
3. **智能解析**：
   - 二进制（如 .jar）：读取内部 mods.toml / fabric.mod.json 提取版本变更。
   - 文本（如 .json, .toml）：保存旧内容，计算文本 Diff。
4. **交互补全**：CLI 展示变动列表，提示用户输入本次更新的"核心摘要/Bug修复"。
5. **日志导出**：合并机器数据与人工摘要，输出 Markdown 格式的 Changelog。

## 3. 核心工作流
1. **扫描与 Hash**：递归扫描目标目录，应用类 `.gitignore` 规则过滤，计算 MD5/SHA256。
2. **差异对比**：与历史快照对比，得出 增(Add)、删(Delete)、改(Modify)、重命名(Rename)。
3. **配置驱动的解析引擎 (核心)**：
   - 摒弃硬编码，所有解析逻辑由 `rules.yaml` 或 `config.json` 驱动。
   - **存副本规则**：按后缀配置哪些文本文件需要保存历史内容以供 Diff。
   - **深潜规则 (Deep Dive)**：配置针对压缩包（如 .jar/.zip）的内部探测路径（如：读取包内 `META-INF/mods.toml` 的 `version` 字段）。
4. **交互补全**：GUI 展示变动列表，提示用户输入本次更新的"核心摘要/Bug修复"。
5. **日志导出**：合并机器数据与人工摘要，输出 Markdown 格式的 Changelog。

## 6. 关键设计原则 (防坑指南)
- **拒绝实时监控**：不做 FileSystemWatcher，只做按需的"快照对比"，避免事件风暴。
- **防假修改**：必须校验 Hash，仅凭时间戳判断不可靠。
- **无硬编码**：所有路径、忽略规则、解析策略均通过配置文件管理。
